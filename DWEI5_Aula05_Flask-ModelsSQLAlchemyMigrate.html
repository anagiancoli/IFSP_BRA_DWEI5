1.1. O que SQLAlchemy ?

  • De acordo com Andrade, Ana Paula (2019), o SQLAlchemy um framework de Mapeamento Objeto-Relacional SQL (ORM) de cdigo aberto sobre a licena MIT.
  • Consulte a Documentao.
  • O Mapeamento uma tcnica de programao que auxilia na converso de dados entre banco de dados relacionais e linguagens de programao que so orientadas a objetos.
  • A utilizao deste framework faz com que o programador reduza a programao de acesso ao banco de dados, desta forma, obtendo uma produtividade significativa no desenvolvimento de suas aplicaes.
    • Cada classe interpretada como uma tabela.
    • Cada linha de uma determinada tabela, junto com seu relacionamento tratada como instncia do objeto relacionado tabela em questo.
    • Com ele, o desenvolvedor no necessita se preocupar com a utilizao de cdigos SQL no desenvolvimento de aplicaes, j que todo acesso ao banco de dados feito utilizando cdigo Python.
    • Alm disso, o SQLAlchemy possui todas as garantias de segurana necessrias para proteger as aplicaes de ataques ao seus bancos de dados, garantindo, assim, a criao de aplicativos simples e seguros.

1.2. O que Migrate ?

  • Consulte a Documentao.
  • O Flask-Migrate uma extenso que lida com migraes de banco de dados SQLAlchemy para aplicativos Flask usando Alembic. As operaes do banco de dados so disponibilizadas por meio da interface de linha de comando do Flask ou por meio da extenso Flask-Script.
  • OBS: Sempre que ocorrer modificao nos modelos, este processo de migrao dever ser executado. Muitas vezes as alteraes no so aplicadas, uma vez j criado o banco de dados. Neste caso, excluir os arquivos de cache, o banco.db criado dentro de projeto, e a pasta migrations. E executar novamente. Para um processo mais correto, efetuar uma pesquisa mais aprofundada para saber outras formas de correo.

2.3. Models

  • Models so os modelos das tabelas que sero criadas no banco de dados. Para criarmos os models, utilizaremos o SQLAlchemy que nos auxiliar na criao dos modelos. E o Migrate, que nos auxiliar na migrao e efetivao da criao do banco de dados.

2.3.1. class db.Model

  • chamada de db.Model, a classe bsica para todos os modelos que utilizam Flask-SQLAlchemy.
  • Ela armazenada na instncia SQLAlchemy que deve ser criada, neste caso, foi criada no arquivo app.py por meio da instruo:
    • db = SQLAlchemy(config.APP).
  • O Flask-SQLAlchemy permite algumas configuraes como opcionais:
    • O nome da tabela definido automaticamente, caso no seja informado.
    • Quando omitido, o nome da tabela derivado do nome da classe convertido em minsculas.
    • Para substituir o nome da tabela, defina o atributo de classe __tablename__ = nometabela.

2.3.2. Tipos de Colunas

2.3.3. Relacionamentos entre modelos um para muitos

  • De acordo com Pallets Projects, os relacionamentos de um para muitos so os mais comuns. 
class Usuario(db.Model):

    # caso queira indicar o nome da tabela
    __tablename__ = 'usuario'

    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(80), unique=True, nullable=False)
    email = db.Column(db.String(120), unique=True, nullable=False)

    # O metodo __repr__ e usado durante o desenvolvimento para retornar o objeto que esta sendo manipulado
    def __repr__(self):
        return '<User %r>' % self.name

class Pessoa(db.Model):

    # caso queira indicar o nome da tabela
     __tablename__ = 'individuo'

    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(50), nullable=False)
    addresses = db.relationship('Endereco', backref='pessoa', lazy=True)


class Endereco(db.Model):

    # caso queira indicar o nome da tabela
    __tablename__ = 'endereco'

    id = db.Column(db.Integer, primary_key=True)
    email = db.Column(db.String(120), nullable=False)
    person_id = db.Column(db.Integer, db.ForeignKey('pessoa.id'), nullable=False)
  • Column utilizado para definir um campo, que ter o mesmo nome indicado na atribuio.
  • Chaves primrias, utilize: primary_key=True.
  • Para valores no obrigatrios, utilize: nullable=False.
  • Para valores nicos, utilize: unique=True.
  • Chaves estrangeiras, utilize: db.ForeignKey('perfil.id').
  • A funo .relationship(), retorna uma nova propriedade que pode fazer vrias coisas. Dentre elas, pode apontar para a classe Endereco e carregar vrios endereos.
  • Ao utilizar backref='', est indicando uma nova forma de declarar uma nova propriedade na classe Endereco.
  • E lazy define quando o SQLALchemy carregar o banco de dados.

2.3.4. Relacionamentos entre modelos muitos para muitos

  • De acordo com Pallets Projects, os relacionamentos de muitos para muitos, ser necessrio definir uma tabela auxiliar que ser usada para o relacionamento. No utilizar a estrutura dos models.
  • Para esta tabela auxiliar, altamente recomendvel no usar um modelo, mas uma tabela real:
tags = db.Table('tags', 
    db.Column('tag_id', db.Integer, db.ForeignKey('tag.id'), primary_key=True),
    db.Column('page_id', db.Integer, db.ForeignKey('page.id'), primary_key=True)
)

class Page(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    tags = db.relationship('Tag', secondary=tags, lazy='subquery', backref=db.backref('pages', lazy=True))

class Tag(db.Model):
    id = db.Column(db.Integer, primary_key=True)

3. Executando a migrao

  • No terminal, ative o ambiente virtual. Caso no tenha instalado, siga os processos definidos nas aulas anteriores:
    • (1) De dentro da pasta projeto, execute: python -m venv venv.
    • (2) Ative o ambiente virtual, digite: source venv/bin/activate, para o SO Windows, consulte as aulas anteriores.
    • (3) Instale as bibliotecas necessrias para o Flask, utilizando o arquivo instalar.sh ou instalar.bat (conforme seu SO).
    • (4) Defina a varivel de ambiente FLASK_ENV=development, indicando que estamos executando no ambiente de desenvolvimento.
    • Observao: Se as etapas 1 e 3 j foram feitas, basta executar as etapas 2 e 4 para dar continuidade.
  • Em seguida, para efetuar a migrao:
    • Inicialize a configurao do migrate, digitando de dentro da pasta projeto e com o ambiente virtual ativo:

python migrate.py db init

  • Ser exibido algo parecido com isso:

python migrate.py db migrate

  • Ser exibido algo parecido com isso:

python migrate.py db migrate

  • Ser exibido algo parecido com isso:

Verifique se o banco foi criado e suas tabelas, abra o software db browser. O nome do banco criado ser o que foi definido no arquivo config.py

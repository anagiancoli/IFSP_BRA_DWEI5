1.1. O que é SQLAlchemy ?¶

  • De acordo com Andrade, Ana Paula (2019), o SQLAlchemy é um framework de Mapeamento Objeto-Relacional SQL (ORM) de código aberto sobre a licença MIT.
  • Consulte a Documentação.
  • O Mapeamento é uma técnica de programação que auxilia na conversão de dados entre banco de dados relacionais e linguagens de programação que são orientadas a objetos.
  • A utilização deste framework faz com que o programador reduza a programação de acesso ao banco de dados, desta forma, obtendo uma produtividade significativa no desenvolvimento de suas aplicações.
    • Cada classe é interpretada como uma tabela.
    • Cada linha de uma determinada tabela, junto com seu relacionamento é tratada como instância do objeto relacionado à tabela em questão.
    • Com ele, o desenvolvedor não necessita se preocupar com a utilização de códigos SQL no desenvolvimento de aplicações, já que todo acesso ao banco de dados é feito utilizando código Python.
    • Além disso, o SQLAlchemy possui todas as garantias de segurança necessárias para proteger as aplicações de ataques ao seus bancos de dados, garantindo, assim, a criação de aplicativos simples e seguros.

1.2. O que é Migrate ?¶

  • Consulte a Documentação.
  • O Flask-Migrate é uma extensão que lida com migrações de banco de dados SQLAlchemy para aplicativos Flask usando Alembic. As operações do banco de dados são disponibilizadas por meio da interface de linha de comando do Flask ou por meio da extensão Flask-Script.
  • OBS: Sempre que ocorrer modificação nos modelos, este processo de migração deverá ser executado. Muitas vezes as alterações não são aplicadas, uma vez já criado o banco de dados. Neste caso, excluir os arquivos de cache, o banco.db criado dentro de projeto, e a pasta migrations. E executar novamente. Para um processo mais correto, efetuar uma pesquisa mais aprofundada para saber outras formas de correção.

2.3. Models¶

  • Models são os modelos das tabelas que serão criadas no banco de dados. Para criarmos os models, utilizaremos o SQLAlchemy que nos auxiliará na criação dos modelos. E o Migrate, que nos auxiliará na migração e efetivação da criação do banco de dados.

2.3.1. class db.Model¶

  • É chamada de db.Model, a classe básica para todos os modelos que utilizam Flask-SQLAlchemy.
  • Ela é armazenada na instância SQLAlchemy que deve ser criada, neste caso, foi criada no arquivo app.py por meio da instrução:
    • db = SQLAlchemy(config.APP).
  • O Flask-SQLAlchemy permite algumas configurações como opcionais:
    • O nome da tabela é definido automaticamente, caso não seja informado.
    • Quando omitido, o nome da tabela é derivado do nome da classe convertido em minúsculas.
    • Para substituir o nome da tabela, defina o atributo de classe __tablename__ = nometabela.

2.3.2. Tipos de Colunas¶

Tipo Descrição significado
Integer um inteiro
String(size) uma string com um comprimento máximo (opcional em alguns bancos de dados, por exemplo, PostgreSQL)
Text algum texto Unicode mais longo
DateTime data e hora expressas como objeto Python datetime
Float armazena valores de ponto flutuante
Boolean armazena um valor booleano
PickleType armazena um objeto Python em conserva
LargeBinary armazena grandes dados binários arbitrários

2.3.3. Relacionamentos entre modelos um para muitos¶

  • De acordo com Pallets Projects, os relacionamentos de um para muitos são os mais comuns. 
class Usuario(db.Model):

    # caso queira indicar o nome da tabela
    __tablename__ = 'usuario'

    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(80), unique=True, nullable=False)
    email = db.Column(db.String(120), unique=True, nullable=False)

    # O metodo __repr__ e usado durante o desenvolvimento para retornar o objeto que esta sendo manipulado
    def __repr__(self):
        return '<User %r>' % self.name

class Pessoa(db.Model):

    # caso queira indicar o nome da tabela
     __tablename__ = 'individuo'

    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(50), nullable=False)
    addresses = db.relationship('Endereco', backref='pessoa', lazy=True)


class Endereco(db.Model):

    # caso queira indicar o nome da tabela
    __tablename__ = 'endereco'

    id = db.Column(db.Integer, primary_key=True)
    email = db.Column(db.String(120), nullable=False)
    person_id = db.Column(db.Integer, db.ForeignKey('pessoa.id'), nullable=False)
  • Column é utilizado para definir um campo, que terá o mesmo nome indicado na atribuição.
  • Chaves primárias, utilize: primary_key=True.
  • Para valores não obrigatórios, utilize: nullable=False.
  • Para valores únicos, utilize: unique=True.
  • Chaves estrangeiras, utilize: db.ForeignKey('perfil.id').
  • A função .relationship(), retorna uma nova propriedade que pode fazer várias coisas. Dentre elas, pode apontar para a classe Endereco e carregar vários endereços.
  • Ao utilizar backref='', está indicando uma nova forma de declarar uma nova propriedade na classe Endereco.
  • E lazy define quando o SQLALchemy carregará o banco de dados.

2.3.4. Relacionamentos entre modelos muitos para muitos¶

  • De acordo com Pallets Projects, os relacionamentos de muitos para muitos, será necessário definir uma tabela auxiliar que será usada para o relacionamento. Não utilizar a estrutura dos models.
  • Para esta tabela auxiliar, é altamente recomendável não usar um modelo, mas uma tabela real:
tags = db.Table('tags', 
    db.Column('tag_id', db.Integer, db.ForeignKey('tag.id'), primary_key=True),
    db.Column('page_id', db.Integer, db.ForeignKey('page.id'), primary_key=True)
)

class Page(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    tags = db.relationship('Tag', secondary=tags, lazy='subquery', backref=db.backref('pages', lazy=True))

class Tag(db.Model):
    id = db.Column(db.Integer, primary_key=True)

from flask import Flask
from flask_sqlalchemy import SQLAlchemy
from flask_script import Manager
from flask_migrate import Migrate, MigrateCommand 
from config import app_active, app_config


config = app_config[app_active]
app = Flask(__name__)
app.config['SQLALCHEMY_DATABASE_URI'] = config.SQLALCHEMY_DATABASE_URI
app.config['SQLALCHEMY_TRACK_MODIFICATIONS'] = config.SQLALCHEMY_TRACK_MODIFICATIONS
db = SQLAlchemy(app)
migrate = Migrate(app, db)
manager = Manager(app) 
manager.add_command('db', MigrateCommand)

3. Executando a migração¶

  • No terminal, ative o ambiente virtual. Caso não tenha instalado, siga os processos definidos nas aulas anteriores:
    • (1) De dentro da pasta projeto, execute: python -m venv venv.
    • (2) Ative o ambiente virtual, digite: source venv/bin/activate, para o SO Windows, consulte as aulas anteriores.
    • (3) Instale as bibliotecas necessárias para o Flask, utilizando o arquivo instalar.sh ou instalar.bat (conforme seu SO).
    • (4) Defina a variável de ambiente FLASK_ENV=development, indicando que estamos executando no ambiente de desenvolvimento.
    • Observação: Se as etapas 1 e 3 já foram feitas, basta executar as etapas 2 e 4 para dar continuidade.
  • Em seguida, para efetuar a migração:
    • Inicialize a configuração do migrate, digitando de dentro da pasta projeto e com o ambiente virtual ativo:

python migrate.py db init¶

  • Será exibido algo parecido com isso:

python migrate.py db migrate¶

1. Aprofundando o uso do SQLAlchemy

  • O SQLAlchemy possui alguns recursos que facilitam o uso de queries em nossos models.
  • Utiliza o conceito de sesso por meio do uso da instncia db, mtodo session, responsvel por efetuar a abertura e fechamento da conexo com o banco de dados. Assim possvel criar -- insert, atualizar -- update, excluir -- delete e pesquisar -- select, sem haver a necessidade de criar as instrues SQL.

1.1. Algumas instrues do SQLAlchemy

  • Para exemplificar, vamos imaginar que temos uma tabela no banco chamada Alunos.

Pesquisa -- select

  • Pesquisando no banco de dados, isto , efetuando um select:
  • O mtodo all() responsvel por retornar todos os registros contidos na tabela Alunos.
  • Atribuio do retorno da query varivel resposta.
resposta = db.session.query(Alunos).all()
  • O mtodo first() responsvel por retornar somente o primeiro registro contido na tabela Alunos.
  • Atribuio do retorno da query varivel resposta.
resposta = db.session.query(Alunos).first()
  • O mtodo close() responsvel por encerrar a conexo com o banco de dados. No deve ser feito atribuio.
db.session.close()
  • A lista retornada com todos os alunos e armazenada na varivel resposta nos permitir acessar seus atributos -- so os campos da tabela no banco de dados. Exemplos:
# Utilizando os ndices
resposta = db.session.query(Alunos).all()
print(resposta[0].id)
print(resposta[0].nome)
print(resposta[0].idade)

for aluno in resposta:

Inserindo -- insert

  • Inserindo no banco de dados, isto , efetuando um insert:
# Mtodo add(): Efetua a preparao para a insero.
db.session.add(self) 

# Mtodo commit(): Efetiva a insero no banco de dados.
db.session.commit() 

# Mtodo rollback(): Desfaz as alteraes caso tenha ocorrido algum erro na insero. 
db.session.rollback()

Atualizando -- update

  • Atualizando no banco de dados, isto , efetuando um update:
# Mtodo filter(): utilizado para filtrar um dado, no caso o id do Aluno. Seria a clsula **where** do SQL.
# Mtodo update(): prepara para a atualizao passando um dicionrio com o nome do campo e valor a serem modificados.
obj = {
    'nome': 'Ana Giancoli',
}
resposta = db.session.query(Aluno).filter(Aluno.id == self.id).update(obj)

# # Mtodo commit(): Efetiva a atualizao no banco de dados.
db.session.commit()

# Mtodo rollback(): Desfaz as alteraes caso tenha ocorrido algum erro na insero. 
db.session.rollback()

Excluindo -- delete

  • Excluindo no banco de dados, isto , efetuando um delete:
# Mtodo filter(): utilizado para filtrar um dado, no caso o id do Aluno. Seria a clsula **where** do SQL.
# Mtodo delete(): prepara para a excluso.

resposta = db.session.query(Aluno).filter(Aluno.id == self.id).delete()

# # Mtodo commit(): Efetiva a excluso no banco de dados.
db.session.commit()

# Mtodo rollback(): Desfaz as alteraes caso tenha ocorrido algum erro na insero. 
db.session.rollback()

Observao

  • Para o mtodo de seleo, necessrio utilizarmos o mtodo close() para encerrar a conexo com o banco.
  • Porm, para os mtodos de insero, atualizao, e excluso, esta instruo no nessria, uma vez que o prprio mtodo commit() encerra a conexo depois de efetivar a devida instruo.

Filtros no SQLAlchemy

  • Os filtros no SQLAlchemy so as clsulas where do SQL e podem ser utilizadas de duas formas:
  • Mtodo filter(): adiciona-se mais de um campo para efetivar o filtro.
  • Mtodo filter_by(): adiciona-se somente um.

Exemplos

# Filtrando por id
resposta = db.session.query(Aluno.id).filter_by(id=self.id).all()

# Filtando por id e idade
resposta = db.session.query(Aluno.id).filter(Aluno.id == self.id).filter(Aluno.idade > 10).all()

# Filtrando string com like
resposta = db.session.query(Aluno).filter(Aluno.nome.like('%Giancoli%')).all()

Observao

  • Existem diversos outros tipos de filtros, para saber mais consulte Object Relational Tutorial.

Funes de agrupamento no SQLAlchemy

  • O SQLAlchemy trabalha com um objeto chamado func que permite utilizarmos diversas funes bsicas por meio dos mtodos disponveis. Para utilizar, lembrar de importar:
    • from sqlalchemy import func

Exemplos

  • func.count(campo): efetua o clculo da quantidade de linhas do campo em questo.
  • func.avg(campo): retorna a mdia calculada do campo especificado.
  • func.sum(campo): retorna a soma da quantidade de linhas do campo especificado.
  • func.max(campo): retorna o valor mximo do campo especificado.
  • func.min(campo): retorna o valor mnimo do campo especificado.
# func.count()
resposta = db.session.query(func.count(Aluno.id)).all()

# func.avg()
resposta = db.session.query(func.avg(Aluno.idade)).all()

# func.sum()
resposta = db.session.query(func.sum(Aluno.id)).all()

# func.max()
resposta = db.session.query(func.max(Aluno.idade)).all()

# func.min()
resposta = db.session.query(func.sum(Aluno.id)).all()

Melhorando as queries

  • Por meio do uso de algumas clusulas do SQL podemos aplicar nas nossa queries.

    • limit(valor): o limite responsvel por limitar a quantidade de linhas que sero retornadas da pesquisa. Necessrio incluir o mtodo all().
    • order_by(campo): responsvel por ordenar os dados de acordo com o campo passado. Deve ser passado o nome do campo como string ou o objeto mais o atributo que representa o campo.
      • Para ordenar em ordem crescente ou descrescente necessrio importar:
        • from sqlalchemy import desc, asc
    • distinct(campo): responsvel por eliminar duplicidades na pesquisa. Para o caso em que desejamos identificar os itens por algum campo sem repetio.
      • Para utilizar o distinct, necessrio importar:
        • from sqlalchemy import distinct
    • group_by(campo): responsvel por agrupar registros pelo campo passado.
    • having(): usada para especificar condies de filtragem em um query que esteja sendo agrupada.
    • join(tabela): usada para unir uma tabela a outra.

  • Outras informaes, consulte Docs SQLAlchemy

# limit(quantidade)
resposta = db.session.query(Aluno).limit(10).all()

# order_by(campo)
# campo
resposta = db.session.query(Aluno).order_by('nome').all()

# objeto + atributo
resposta = db.session.query(Aluno).order_by(Aluno.nome).all()

# Ordem crescente  o default
resposta = db.session.query(Aluno).order_by(asc(Aluno.nome)).all()

# Ordem descrescente
resposta = db.session.query(Aluno).order_by(desc(Aluno.nome)).all()

# Distinct
# O retorno desta pesquisa ser separado por idades
resposta = db.session.query(distinct(Aluno.idade)).all()

# Group_by
resposta = db.session.query(Aluno.idade).group_by(Aluno.idade).all()

# Ou
resposta = db.session.query(Aluno.idade).group_by('idade').all()

# Having
resposta = db.session.query(Aluno.idade, func.sum(Aluno.idade)).group_by(Aluno.idade).having(func.sum(Aluno.idade) > 21).all()

# Join
resposta = db.session.query(Aluno, Endereco).join(Endereco).filter(Aluno.enderecoid == Endereco.id).all()

Observao

  • O SQLAlchemy permite tambm a criao de queries convencionais. Utiliza-se o mtodo padro **execute().
    • alunos = db.session.execute('SELECT * FROM aluno;')

    # Mtodo adicionado para salvar os componentes.
    # Mtodo .add(): adiciona um objeto na sesso.
    # Mtodo .commit(): efetiva a insero no banco de dados.
    # Mtodo .roolback(): acionado caso ocorra erro na insero,  desfeito qualquer alterao proposta.
    # No se utiliza o mtodo .close(), pois o prprio commit() encerra a conexo aps a alterao.

            db.session.commit() 
            return True
        except Exception as e:
            print(e)
            db.session.rollback()
            return False

2.3. Criando o controllers/Componente.py

  • Localize o arquivo Componente.py dentro da pasta controllers.
  • Este arquivo responsvel por acessar os atributos da model Componente, os quais armazenaro os valores vindos da view definida no arquivo app.py.
# Importando os mtodos datetime para gravar a data e hora no formato adequado
from datetime import datetime

# Importando o model do Componente
from models.Componente import Componente

# Criando uma classe Controller do Componente e definindo seu construtor
class ComponenteController(): 

    # Construtor do Controller
    def __init__(self):

        # Instncia do Componente atribuda ao atributo componente_model
        self.componente_model = Componente()


    # Mtodo que passar para a model Componente, os dados vindo da view
    def save_componente(self, obj):
        self.componente_model.sigla = obj['sigla'] 
        self.componente_model.descricao = obj['descricao'] 
        self.componente_model.qtdeaulas = obj['qtdeaulas']
        self.componente_model.date_created = datetime.now() 
        self.componente_model.status = 1 
        self.componente_model.user_created = obj['user_created'] 
        self.componente_model.areaid = obj['areaid'] 
        self.componente_model.moduloid = obj['moduloid'] 

        # Mtodo para salvar os componentes, executar o mtodo definido na model Componente.py
        return self.componente_model.save()

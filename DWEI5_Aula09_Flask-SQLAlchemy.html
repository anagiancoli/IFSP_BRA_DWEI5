1. Aprofundando o uso do SQLAlchemy¶

  • O SQLAlchemy possui alguns recursos que facilitam o uso de queries em nossos models.
  • Utiliza o conceito de sessão por meio do uso da instˆncia db, método session, responsável por efetuar a abertura e fechamento da conexão com o banco de dados. Assim é possível criar -- insert, atualizar -- update, excluir -- delete e pesquisar -- select, sem haver a necessidade de criar as instruções SQL.

1.1. Algumas instruções do SQLAlchemy¶

  • Para exemplificar, vamos imaginar que temos uma tabela no banco chamada Alunos.

Pesquisa -- select¶

  • Pesquisando no banco de dados, isto é, efetuando um select:
  • O método all() é responsável por retornar todos os registros contidos na tabela Alunos.
  • Atribuição do retorno da query à variável resposta.
resposta = db.session.query(Alunos).all()
  • O método first() é responsável por retornar somente o primeiro registro contido na tabela Alunos.
  • Atribuição do retorno da query à variável resposta.
resposta = db.session.query(Alunos).first()
  • O método close() é responsável por encerrar a conexão com o banco de dados. Não deve ser feito atribuição.
db.session.close()
  • A lista retornada com todos os alunos e armazenada na variável resposta nos permitirá acessar seus atributos -- são os campos da tabela no banco de dados. Exemplos:
# Utilizando os índices
resposta = db.session.query(Alunos).all()
print(resposta[0].id)
print(resposta[0].nome)
print(resposta[0].idade)

...

# Ou ainda utilizando a estrutura de repetição for

for aluno in resposta:
    print(aluno.id)
    print(aluno.nome)
    print(aluno.idade)

...
  • O select poderá ser feito somente selecionando algumas colunas ou campos. Exemplos:
# Utilizando o método first()
alunos = db.session.query(Aluno.id, Aluno.nome, Aluno.idade).first()

alunos.id
alunos.nome
alunos.idade

...


# Ou utilizando o método all()
alunos = db.session.query(Aluno.id, Aluno.nome, Aluno.idade).first()

for aluno in alunos:
    print(aluno.id)
    print(aluno.nome)
    print(aluno.idade)
...

Inserindo -- insert¶

  • Inserindo no banco de dados, isto é, efetuando um insert:
# Método add(): Efetua a preparação para a inserção.
db.session.add(self) 

# Método commit(): Efetiva a inserção no banco de dados.
db.session.commit() 

# Método rollback(): Desfaz as alterações caso tenha ocorrido algum erro na inserção. 
db.session.rollback()

Atualizando -- update¶

  • Atualizando no banco de dados, isto é, efetuando um update:
# Método filter(): utilizado para filtrar um dado, no caso o id do Aluno. Seria a clásula **where** do SQL.
# Método update(): prepara para a atualização passando um dicionário com o nome do campo e valor a serem modificados.
obj = {
    'nome': 'Ana Giancoli',
}
resposta = db.session.query(Aluno).filter(Aluno.id == self.id).update(obj)

# # Método commit(): Efetiva a atualização no banco de dados.
db.session.commit()

# Método rollback(): Desfaz as alterações caso tenha ocorrido algum erro na inserção. 
db.session.rollback()

Excluindo -- delete¶

  • Excluindo no banco de dados, isto é, efetuando um delete:
# Método filter(): utilizado para filtrar um dado, no caso o id do Aluno. Seria a clásula **where** do SQL.
# Método delete(): prepara para a exclusão.

resposta = db.session.query(Aluno).filter(Aluno.id == self.id).delete()

# # Método commit(): Efetiva a exclusão no banco de dados.
db.session.commit()

# Método rollback(): Desfaz as alterações caso tenha ocorrido algum erro na inserção. 
db.session.rollback()

Observação¶

  • Para o método de seleção, é necessário utilizarmos o método close() para encerrar a conexão com o banco.
  • Porém, para os métodos de inserção, atualização, e exclusão, esta instrução não é nessária, uma vez que o próprio método commit() encerra a conexão depois de efetivar a devida instrução.

Filtros no SQLAlchemy¶

  • Os filtros no SQLAlchemy são as clásulas where do SQL e podem ser utilizadas de duas formas:
  • Método filter(): adiciona-se mais de um campo para efetivar o filtro.
  • Método filter_by(): adiciona-se somente um.

Exemplos¶

# Filtrando por id
resposta = db.session.query(Aluno.id).filter_by(id=self.id).all()

# Filtando por id e idade
resposta = db.session.query(Aluno.id).filter(Aluno.id == self.id).filter(Aluno.idade > 10).all()

# Filtrando string com like
resposta = db.session.query(Aluno).filter(Aluno.nome.like('%Giancoli%')).all()

Observação¶

  • Existem diversos outros tipos de filtros, para saber mais consulte Object Relational Tutorial.

Funções de agrupamento no SQLAlchemy¶

  • O SQLAlchemy trabalha com um objeto chamado func que permite utilizarmos diversas funções básicas por meio dos métodos disponíveis. Para utilizar, lembrar de importar:
    • from sqlalchemy import func

Exemplos¶

  • func.count(campo): efetua o cálculo da quantidade de linhas do campo em questão.
  • func.avg(campo): retorna a média calculada do campo especificado.
  • func.sum(campo): retorna a soma da quantidade de linhas do campo especificado.
  • func.max(campo): retorna o valor máximo do campo especificado.
  • func.min(campo): retorna o valor mínimo do campo especificado.
# func.count()
resposta = db.session.query(func.count(Aluno.id)).all()

# func.avg()
resposta = db.session.query(func.avg(Aluno.idade)).all()

# func.sum()
resposta = db.session.query(func.sum(Aluno.id)).all()

# func.max()
resposta = db.session.query(func.max(Aluno.idade)).all()

# func.min()
resposta = db.session.query(func.sum(Aluno.id)).all()

Melhorando as queries¶

  • Por meio do uso de algumas cláusulas do SQL podemos aplicar nas nossa queries.

    • limit(valor): o limite é responsável por limitar a quantidade de linhas que serão retornadas da pesquisa. Necessário incluir o método all().
    • order_by(campo): é responsável por ordenar os dados de acordo com o campo passado. Deve ser passado o nome do campo como string ou o objeto mais o atributo que representa o campo.
      • Para ordenar em ordem crescente ou descrescente é necessário importar:
        • from sqlalchemy import desc, asc
    • distinct(campo): é responsável por eliminar duplicidades na pesquisa. Para o caso em que desejamos identificar os itens por algum campo sem repetição.
      • Para utilizar o distinct, é necessário importar:
        • from sqlalchemy import distinct
    • group_by(campo): é responsável por agrupar registros pelo campo passado.
    • having(): é usada para especificar condições de filtragem em um query que esteja sendo agrupada.
    • join(tabela): é usada para unir uma tabela a outra.

  • Outras informações, consulte Docs SQLAlchemy

# limit(quantidade)
resposta = db.session.query(Aluno).limit(10).all()

# order_by(campo)
# campo
resposta = db.session.query(Aluno).order_by('nome').all()

# objeto + atributo
resposta = db.session.query(Aluno).order_by(Aluno.nome).all()

# Ordem crescente é o default
resposta = db.session.query(Aluno).order_by(asc(Aluno.nome)).all()

# Ordem descrescente
resposta = db.session.query(Aluno).order_by(desc(Aluno.nome)).all()

# Distinct
# O retorno desta pesquisa será separado por idades
resposta = db.session.query(distinct(Aluno.idade)).all()

# Group_by
resposta = db.session.query(Aluno.idade).group_by(Aluno.idade).all()

# Ou
resposta = db.session.query(Aluno.idade).group_by('idade').all()

# Having
resposta = db.session.query(Aluno.idade, func.sum(Aluno.idade)).group_by(Aluno.idade).having(func.sum(Aluno.idade) > 21).all()

# Join
resposta = db.session.query(Aluno, Endereco).join(Endereco).filter(Aluno.enderecoid == Endereco.id).all()

Observação¶

  • O SQLAlchemy permite também a criação de queries convencionais. Utiliza-se o método padrão **execute().
    • alunos = db.session.execute('SELECT * FROM aluno;')

    # Método adicionado para salvar os componentes.
    # Método .add(): adiciona um objeto na sessão.
    # Método .commit(): efetiva a inserção no banco de dados.
    # Método .roolback(): acionado caso ocorra erro na inserção, é desfeito qualquer alteração proposta.
    # Não se utiliza o método .close(), pois o próprio commit() encerra a conexão após a alteração. 
    # Na pesquisa precisamos fechar, pois não usamos o commit().
    def save(self):
        try:
            db.session.add(self) 
            db.session.commit() 
            return True
        except Exception as e:
            print(e)
            db.session.rollback()
            return False

2.3. Criando o controllers/Componente.py¶

  • Localize o arquivo Componente.py dentro da pasta controllers.
  • Este arquivo é responsável por acessar os atributos da model Componente, os quais armazenarão os valores vindos da view definida no arquivo app.py.
# Importando os métodos datetime para gravar a data e hora no formato adequado
from datetime import datetime

# Importando o model do Componente
from models.Componente import Componente

# Criando uma classe Controller do Componente e definindo seu construtor
class ComponenteController(): 

    # Construtor do Controller
    def __init__(self):

        # Instância do Componente atribuída ao atributo componente_model
        self.componente_model = Componente()


    # Método que passará para a model Componente, os dados vindo da view
    def save_componente(self, obj):
        self.componente_model.sigla = obj['sigla'] 
        self.componente_model.descricao = obj['descricao'] 
        self.componente_model.qtdeaulas = obj['qtdeaulas']
        self.componente_model.date_created = datetime.now() 
        self.componente_model.status = 1 
        self.componente_model.user_created = obj['user_created'] 
        self.componente_model.areaid = obj['areaid'] 
        self.componente_model.moduloid = obj['moduloid'] 

        # Método para salvar os componentes, executará o método definido na model Componente.py
        return self.componente_model.save()

1. Estrutura organizacional do Projeto

1.1. Criando nosso Projeto

  • A partir desta aula, iremos criar um projeto implementando diversas bibliotecas de terceiros agregadas ao Flask.

  • A ideia deste projeto surgiu aps a leitura do livro: [Autor: Tiago Silva] Flask de A a Z, Crie aplicaes web mais completas e robustas em Python, da editora Casa do Cdigo.

  • Crie uma pasta nomeada:

    • projeto.
  • Dentro da pasta projeto, crie os arquivos nomeados:
    • app: conter as informaes principais para iniciar o projeto.
    • config.py: conter as informaes de configurao bsica do app.
    • migrate.py: conter as informaes necessrias para criarmos nosso banco de dados por meio de migrao. Ser executado uma nica vez, caso tenha alterao de campos e tabelas no banco, dever ser executado novamente.
    • run.py: ser utilizado para executar o app.
  • Dentro da pasta projeto, crie as pastas:
    • admin: conter os os arquivos com as views e rotas do administrador da aplicao.
    • controllers: conter os arquivos com as rotas configuradas.
    • models: conter os modelos das classes a serem criadas no banco de dados.
    • static: conter os arquivos estticos, tal como css, js etc.
    • templates: conter os arquivos html das pginas.
  • Dentro da pasta admin, crie dois arquivos com extenso py:
    • Admin.py
    • Views.py
  • Dentro da pasta controllers, crie dois arquivos com extenso py:
    • Email.py
    • Usuario.py
  • Dentro da pasta models, crie dois arquivos com extenso py:
    • Area.py
    • Componente.py
    • Modulo.py
    • Perfil.py
    • Usuario.py
  • Dentro da pasta static, crie duas pastas:
    • css
    • js
    • Dentro da pasta css, crie dois arquivos com extenso css:
      • home.css
      • login.css
    • Dentro da pasta js, crie um arquivo com extenso js:
      • custom.js

  • Dentro da pasta templates, crie dois arquivos com extenso html:

    • admin.html
    • login.html
    • recuperar.py
    • trocar_senha.py

4.1. Requisitos e Regras de Negcio do nosso Projeto

  • Seguindo os conceitos adquiridos, vamos criar um novo projeto na estrutura MVC. Importante salientar que h diversas formas de criao, o que depender do programador ou da especificao de cada projeto.

  • Requisitos e Regras de Negcio:

    • O Administrador dever ter permisso em todas as funcionalidades do aplicativo.
    • O Professor poder criar, atualizar, listar e excluir as reas, mdulos, componentes curriculares.
    • O Aluno poder apenas visualizar os componentes curriculares, alm do seu prprio perfil.
    • O Visitante somente consulta nos componentes curriculares.

  • Views:

    • Login: sero criadas as telas de login e recuperao de senha.
    • Usurios: sero criadas as telas de listagem de usurios, bem como as telas de edio, excluso e criao de novos usurios.
    • Perfis: sero criadas as telas de acesso s configuraes dos perfis dos tipos de perfis de acesso.
    • reas: sero criadas as telas de listagem de reas, bem como as telas de edio, excluso e criao de novas reas.
    • Mdulos: sero criadas as telas de listagem de mdulos, bem como as telas de edio, excluso e criao de novos mdulos.
    • Componentes: sero criadas as telas de listagem de componentes, bem como as telas de edio, excluso e criao de novos componentes curriculares.

1.1. Ambiente Virtual e instalaes

  • Lembre de criar o ambiente virtual venv dentro da pasta projeto.
  • Para isso, de dentro da pasta projeto, execute:
    • python3 -m venv venv. (Obs.: Caso j tenha feito, no h necessidade de executar novamente, basta ativar o ambiente virtual.)

  • Em seguida, inicie, ou melhor, ative o ambiente virtual venv, por meio:

    • source venv/bin/activate, conforme seu SO.

  • Faa a instalao do Flask e demais dependncias por meio da execuo do arquivo criado na primeira aula: instalar.bat ou instalar.sh. (IMPORTANTE: ambiente venv precisa estar ativo!!).

  • Consulte: DWEI5_Aula01_IntroducaoFlask.html
  • Para executar:
    • Criar uma nica vez e depois replicar nas pastas quando for necessrio.
    • No Windows, criar o arquivo instalar.bat por meio de um editor de texto, e ao salvar, substituir .txt por .bat. (Comando de execuo de dentro do ambiente virtual ativo: dois cliques sobre instalar.bat)
    • No MacOS, criar o arquivo instalar.sh por meio de um editor de texto, e ao salvar, substituir .txt por .sh. (Comando de execuo de dentro do ambiente virtual ativo: bash instalar.sh)

import os
import random, string


# Classe Config criada com as configuraes do Projeto.
class Config(object):
    # Permite o uso de criptografia em sesses do Flask.
    CSRF_ENABLED = True

    # Palavra-chave secreta utilizada para criptografar chaves e valores, como por exemplo: senhas.
    SECRET = 'ysb_92=qe#dgjf8%0ng+a*#4rt#5%3*4kw5%i2bck*gn@w3@f&-&'

    # Caminho completo onde os templates estaro disponveis no projeto.
    # Para capturar o endereco do arquivo atual: os.path.dirname(__file__)
    # Para capturar o caminho absoluto: os.path.abspath(variavel)
    # Para unir as strings: os.path.join(str1, str2)
    TEMPLATE_FOLDER = os.path.join(os.path.dirname(os.path.abspath(__file__)), 'templates')

    # Identificao do caminho completo do local do projeto
    ROOT_DIR = os.path.dirname(os.path.abspath(__file__)) 

    # Constante inicializada com None, posteriormente ao criar o app, esta ser atribuda.
    APP = None

    # Varivel com o endereo do banco de dados. O mtodo .join  utilizado para 
    # concatenar o nome completo contendo o endereo do banco de dados a ser utilizado pelo app.
    # O banco de dados ser chamado de app.db. Pode ser utilizado qualquer nome.
    SQLALCHEMY_DATABASE_URI = 'sqlite:///' + os.path.join(ROOT_DIR, 'app.db')

    # CHAVE API COPIADA DO SITE - UNICA PARA CADA UM ** OMITIR ANTES DE DIVULGAR CODIGO ***
    # Inserida para enviar email por meio do servio sendgrid. (veremos mais adiante)
    SENDGRID_API_KEY = 'Copiar a chave do site'

# Classe Development herda da classe Config, e foi criada com as configuraes do Projeto para ambiente de Desenvolvimento.
class Development(Config):
    # Habilitando o modo de teste.
    TESTING = True

    # Habilitando o modo de debug.
    DEBUG = True

    # Definindo o IP do servidor Flask.
    IP_HOST = 'localhost'

    # Definindo a PORTA de acesso do servidor Flask.
    PORT_HOST = 8000

    # Definindo a URL root da aplicao Flask.
    URL_MAIN = 'http://%s:%s/' % (IP_HOST, PORT_HOST)


# Classe TestingConfig herda da classe Config, e foi criada com as configuraes do Projeto para ambiente de Teste.
class TestingConfig(Config):
    TESTING = True
    DEBUG = True

    # Aqui geralmente  um IP de um servidor na nuvem e no o endereo da mquina local.
    IP_HOST = 'localhost' 

    PORT_HOST = 5000
    URL_MAIN = 'http://%s:%s/' % (IP_HOST, PORT_HOST)


# Classe Production herda da classe Config, e foi criada com as configuraes do Projeto para ambiente de Produo. 
class Production(Config):
    DEBUG = False
    TESTING = False

    # Aqui geralmente  um IP de um servidor na nuvem e no o endereo da mquina local.
    IP_HOST = 'localhost' 

    PORT_HOST = 8080
    URL_MAIN = 'http://%s:%s/' % (IP_HOST, PORT_HOST)

# Varivel com as 3 configuraes dos ambientes. Ser utilizada para saber qual ambiente estamos utilizando.   
app_config = {
    'development': Development(), 
    'testing': TestingConfig(), 
    'production': Production()
}

# Varivel que receber o valor do ambiente que est ativo e em uso. O mtodo os.getenv('NOME_DA_CHAVE'). Onde NOME_DA_CHAVE  o nome da varivel de ambiente, neste caso FLASK_ENV. Esta varivel receber o tipo de ambiente que est sendo utilizado antes da execuo do arquivo run.py. Deve ser informada uma nica vez. Porm todas as vezes que o ambiente virtual for ativado. E sempre antes de exeutar o run.py. 
app_active = os.getenv('FLASK_ENV')

2.2. Editando o arquivo app.py

  • Localize o arquivo app.py dentro da pasta projeto.
  • Abra e insira as instrues.
  • No prprio arquivo foram inseridos o significado de cada instruo.
# Ao inserir esta primeira linha (# -*- coding: utf-8 -*-) no arquivo, estamos indicando que o mesmo deve utilizar a codificao utf-8, no caso para nosso idioma que possui acentos.

# -*- coding: utf-8 -*-

#  importado da biblioteca flask, a classe Flask.
from flask import Flask

#  importado do arquivo config.py, as variveis app_config, app_active que app_config: possui as configuraes e o tipo de ambiente a ser utilizado (Desenvolvimento, Teste ou Producao). E app_active: possui as configuraes de qual ambiente est sendo utilizado por meio da varivel de ambiente FLASK_ENV.
from config import app_config, app_active

# A varivel config recebe a atribuio do ambiente ativo.
config = app_config[app_active] 

# Mtodo create_app que recebe como argumento com todas as configuraes da aplicao.
def create_app(config_name):

    # A varivel app recebe uma instncia de Flask, passando a configurao da localizao dos templates. Pode 
    # receber diversas configuraes.
    app = Flask(__name__, template_folder='templates')

    # O atributo secret_key da aplicao app, recebe a configurao da chave secreta do arquivo config.py, por meio da varivel config e atributo SECRET.
    app.secret_key = config.SECRET 

    # Efetua o carregamento do arquivo config.py
    app.config.from_object(app_config[config_name]) 
    app.config.from_pyfile('config.py')

    # Primeira rota a ser criada. A rota principal root.
    # Quando se cria duas ou mais rotas seguidas antes do mtodo, isso indica ao navegador que qualquer uma delas acessa o mtodo a seguir. Neste exemplo tanto http://localhost:8000/ e http://localhost:8000/login/ acessam o mdtodo index(). Que retorna uma string.
    @app.route('/') 
    @app.route('/login/')
    def index():
        return 'Meu Primeiro APP Flask!' 


    # Retorno do mtodo create_app(). Retorna a instncia do app criada.
    return app

2.3. Editando o arquivo run.py

  • Localize o arquivo run.py dentro da pasta projeto.
  • Abra e insira as instrues.
  • No prprio arquivo foram inseridos o significado de cada instruo.
# Importa a biblioteca que permite a execuo de intrues no sistema.
import sys

# do pacote app.py importa o mtodo create_app, o qual criar uma instncia da aplicao.
from app import create_app

# do mdulo config importa o as variveis app_config, app_active que possuem as configuraes do ambiente.
from config import app_config, app_active

# Importa da biblioteca importlib, o mtodo reload para que a cada alterao em algum arquivo, o servidor Flask seja reiniciado.
from importlib import reload

# A varivel config foi criada para receber o ambiente ativo.
config = app_config[app_active] 

# A varivel config.APP, APP constante definida no arquivo config.py foi criada para receber a aplicao ativa. A aplicao ativa,  definida no arquivo app.py e importada pelo run.py.
config.APP = create_app(app_active)

# Verificar se o arquivo run.py est sendo executado de forma direta ou no. Se for importado, o if no ser executado. Para que seja executado: no terminal digitar python run.py de dentro da pasta projeto.
# Ao ser executado diretamente run.py, ser iniciado o servidor Flask. com as configuraes definidas no arquivo config.py para o ip do host e porta.
if __name__ == '__main__':
    config.APP.run(host=config.IP_HOST, port=config.PORT_HOST) 
    # executado sempre que ocorrer alguma alterao em um arquivo do projeto.
    reload(sys)

1.1. Apresentação JINJA¶

  • De acordo com Gaspar, D. e Stouffer, J. em Mastering Flask Web Development, Jinja é uma linguagem de templates (modelos) escrita em Python.
  • Uma linguagem de modelo é projetada em um formato simples para ajudar a automatizar a criação de documentos. Em qualquer linguagem de template, as variáveis passadas para o template substituem elementos predefinidos no template.
  • No Jinja, as substituições de variáveis são definidas por {{ variavel }}. A sintaxe {{ variavel }} é chamada de bloco variável.
  • Outro tipo de bloco, são os blocos de controle definidos por {% controle %} que declaram funções de linguagem, como loops ou instruções if.
  • Exemplo:
<!-- Exemplo template base.html -->
<!DOCTYPE html>
<html>
<head>
    <title>
        {% block titulo %}
            Template Base
        {% endblock %}
    </title>
</head>
<body>
    <div>
       {% block conteudo %}

       {% endblock %}
    </div>
</body>
</html>

<!-- Exemplo de index.html -->
{% extends "base.html" %}

{% block titulo %} 
    Template Index 
{% endblock %}

{% block conteudo %}
    <h1> {{ title }} </h1>
    <ul>
    {% for usuario in usuarios %}
        <li><a href="{{ usuario.url }}">{{ usuario.apelido }}</a></li>
    {% endfor %}
    </ul>
{% endblock %}

O que significa ?¶

<!-- Exemplo de index.html -->
<!-- Quando utilizamos extends e o nome do arquivo HTML entre aspas, estamos indicando ao Jinja
 para herdar o template base.html -->
{% extends "base.html" %}  

<!-- Ao utilizarmos blocos, e identificarmos o bloco com o nome titulo. Para substituir o titulo 
do template base.html, precisamos especificar desta forma. -->
{% block titulo %} Template Index {% endblock %}
{% block conteudo %} O que for escrito aqui substituirá o body do template base.html {% endblock %}
{{ title }} será substituído pelo conteúdo da variável title.

2. Continuando a criação da estrutura organizacional do projeto¶

  • Crie uma pasta nomeada:
    • projeto.
  • Dentro da pasta projeto, crie uma pasta nomeada:
    • app.
  • Dentro da pasta app, crie as pastas:
    • templates.
  • Dentro da pasta templates, crie dois arquivos com extensão html:
    • base.html.
    • index.html.
  • Lembre de criar o ambiente virtual venv dentro da pasta projeto. Para isso, de dentro da pasta projeto, execute: python3 -m venv venv. (Obs.: Caso já tenha feito, não há necessidade de executar novamente, basta ativar o ambiente virtual.)
  • Em seguida, inicie, ou melhor, ative o ambiente virtual venv, por meio: source venv/bin/activate, conforme seu SO.
  • Faça a instalação do Flask. (Obs.: uma vez feita dentro do ambiente virtual, não é necessário a instalação novamente quando for utilizar o ambiente. Basta uma única vez.)

  • Inicie o servidor flask executando de dentro da pasta app: python3 __init__.py. (Obs: esta execução é feita desta forma devido a criação do nosso arquivo inicial.)

4. Editando o arquivo __init__.py¶

  • Os modelos de renderização são criados pelo Flask por meio do mecanismo de templates Jinja para gerar os arquivos html.
  • Para renderizar, utilize:
    • render_template("nometemplate.html", variavel=variavelpassada, ..., ...)
    • Pode conter quantas variáveis forem necessárias para renderizar o modelo ou não enviar nenhuma.
# Importando o flask
from flask import Flask

# Para renderizar o template
from flask import render_template, request   <--------- Adicionado

# __name__ : indica o pacote onde a aplicacao esta.
app = Flask(__name__)

# Criando a primeira rota, no caso, a raiz da aplicacao
# Quando criamos rotas seguidas uma da outra, elas acessam a 
# mesma requisicao HTTP.
@app.route("/")
@app.route("/index")

# Criamos uma funcao com mesmo nome da rota para manter padrao
# e facilitar o entendimento
def index():
    return render_template("index.html")     <--------- Modificado

# Verificamos se o arquivo app.py está sendo executado pelo terminal e, 
# caso positivo, iniciamos o servidor do Flask.
if __name__ == "__main__":
    app.run()

2.3. Utilizando o Protocolo HTTP para requisições¶

  • De acordo com [Silva, Tiago] no livro Flask de A a Z, é fundamental para que a aplicação faça a comunicação entre o cliente e o servidor, o navegador envia uma solicitação para o servidor com as informações que serão processadas nele e, aguarda uma resposta. Após o processamento dos resultados, a resposta é devolvida ao navegador (cliente).
  • O Protocolo HTTP executa diversas requisições que são responsáveis por indicar qual método está sendo executado.
    • Método GET: requisições utilizadas em pesquisas, consultas. Informações explícitas na URL do navegador.
    • Método POST: requisições enviadas ao servidor com opção de modificação.
    • Método PUT: requisições enviadas ao servidor para atualização.
    • Método DELETE: requisições enviadas ao servidor para exclusão.
    • Método HEAD: requisições similares ao GET, porém, sem o corpo das informações. Não são explícitas.
    • Método PATCH: similar ao PUT, aplica modificações no servidor, porém, partes de um recurso.

2.3. Utilizando o Protocolo HTTP para requisies

  • De acordo com [Silva, Tiago] no livro Flask de A a Z, fundamental para que a aplicao faa a comunicao entre o cliente e o servidor, o navegador envia uma solicitao para o servidor com as informaes que sero processadas nele e, aguarda uma resposta. Aps o processamento dos resultados, a resposta devolvida ao navegador (cliente).
  • O Protocolo HTTP executa diversas requisies que so responsveis por indicar qual mtodo est sendo executado.
    • Mtodo GET: requisies utilizadas em pesquisas, consultas. Informaes explcitas na URL do navegador.
    • Mtodo POST: requisies enviadas ao servidor com opo de modificao.
    • Mtodo PUT: requisies enviadas ao servidor para atualizao.
    • Mtodo DELETE: requisies enviadas ao servidor para excluso.
    • Mtodo HEAD: requisies similares ao GET, porm, sem o corpo das informaes. No so explcitas.
    • Mtodo PATCH: similar ao PUT, aplica modificaes no servidor, porm, partes de um recurso.

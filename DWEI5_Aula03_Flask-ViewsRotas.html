1.1. Apresentao VIEWS e ROTAS

  • As views so funes, ou classes que respondem por uma determinada URL.
  • A funo da view capturar os parmetros enviados pelo cliente por meio da URL, e efetuar o processamento com o objetivo de responder solicitao, com algum tipo de contedo, mensagem de status, texto, template renderizado json etc.

  • Por exemplo:

    • http://localhost:8000/noticias/brasil?categoria=policia&quantidade=2

    • O que significa ?

      • /noticias/: o endpoint mapeado pela view.
      • /brasil/: o argumento passado para a view.

      • categoria=policia&quantidade=2: so os parmetros (categoria e quantidade) passados para a view.

      • O Flask faz boa parte do roteamento por meio do @app.route().

      • Durante o desenvolvimento da aplicao Flask, interessante utilizar alguns parmetros para agilizar a depurao e codificao. Exemplo:
        • Ao invs de usar simplesmente app.run(). Utilize:
        • app.run(debug=True, use_reloader=True): repectivamente, o atributo debug auxilia na identificao do erro, caso ocorra. E o atributo use-reloader, facilita reiniciar o servidor flask, sempre que ocorrer alguma alterao nos arquivos. Ao lanar em produo, altere para False.

2. Tipos de roteamento

2.1. Roteamento no explcito

  • A vantagem de utilizar .route() para rotear que o Flask usar o nome da funo automaticamente como endpoint.
  • Exemplo: Neste caso, o endpoint o lista_de_noticias.
from flask import Flask, render_template, request

app = Flask(__name__)

@app.route("/noticias/<pais>")
def lista_de_noticias(pais):
    cat = request.args.get("categoria")
    qtd = request.args.get("quantidade")
    noticias = BD.query(pais=pais, categoria=cat).limit(qtd)
    return render_template("lista_de_noticias", noticias=noticias), 200

app.run(debug=True, use_reloader=True)

2.2. Roteamento explcito

  • Muito til quando necessita automatizar ou centralizar o mapeamento de urls em um nico local do projeto. a maneira utilizada pelos Blueprints.
  • Desvantagens: h necessidade de informar explicitamente cada endpoint e a funo da view a ser mapeada.
  • Exemplo:

2.3. Regras de URLs

  • De acordo com Rocha, Bruno, as urls so recebidas no formato texto.

    • Por exemplo: /noticias/1.

      • Em alguns casos queremos que o valor passado como argumento seja convertido para um tipo de dados especfico, ou seja, queremos receber o 1 como um inteiro.
      • O Flask utiliza como base a ferramenta Werkzeug que internamente utiliza o .route() e alguns conversores para as regras de URLs.

      • Tipos de conversores: < converter : variavel >

        • string: @app.route("/user/< username >")
        • int: @app.route("/user/< int:id >")
        • float: @app.route("/user/< float:dollar >")
        • path: @app.route("/user/< path:subpath >")

      • Os conversores padres so:

        • /noticias/< categoria >, recebe o parmetro "categoria" no formato unicode, exemplo "entretenimento".
        • /noticias/< int:noticia_id >, recebe o parmetro "noticia_id" no formato inteiro, exemplo: "1".
        • /cotacao/< float:dolar >/, recebe o parmetro "dolar" como float, exemplo: "3.2".
        • /imagem/< path >, recebe o parmetro "path" como um caminho, exemplo: "animais/marssupiais/quokka.png".
        • Tambm possivel utilizar regex, passar parmetros para validao ou at mesmo criar seus prprios conversores de url.

Observao IMPORTANTE:

  • Se declarar a url sem a barra no final e o usurio tentar acessar, somente conseguir se no utilizar a barra no final.
  • Se declarar a url com a barra no final e o usurio tentar acessar, mesmo sem indicar a barra, o Flask adiciona automaticamente.

2.4. Construo de URLs

  • Podemos construir URLs para uma funo especfica, utilizando: url_for( ).
  • Por padro, aceita o nome da funo como 1o. argumento e qualquer outro nmero de argumentos seguidos de combinaes palavra-chave, onde cada par refere-se a uma parte varivel da regra da URL.
<!-- Arquivo templates/base.html -->

<a href="{{ url_for('controls.profile') }}" class="navbar-item">

3. Mtodos HTTP

  • Os aplicativos da web utilizam diversas formas de acesso por meio de URLs e mtodos HTTP.
  • Por padro, no Flask uma rota responde apenas a solicitaes GET.
  • Porm, pode-se utilizar o argumento METHODS nas rotas criadas por .route() para lidar com diferentes mtodos de acesso.
  • Por exemplo:
from flask import request

@app.route('/login', methods=['GET', 'POST'])
def login():
    if request.method == 'POST':
        return do_the_login()
    else:
        return show_the_login_form()

3.1. Arquivos estticos

  • Para utilizar arquivos de extenses css, js e outros tipos estticos, utiliza-se a criao de uma pasta nomeada static dentro da pasta app, ao lado da pasta templates.
  • Para gerar as URLs dos arquivos estticos, utiliza-se:
    • url_for('static', filename='style.css')
  • E o arquivo style.css, deve ser inserido na pasta static.
  • O Flask utiliza o mecanismo de Template Jinja2 para gerar os arquivos HTML.
  • Para renderizar um template, utilize: render_template('nomeTemplete.html'). Onde que nomeTemplete.html deve estar dentro da pasta templates.
  • possvel passar variveis para os templates, veremos mais adiante.

from flask import Flask

app = Flask(__name__)

# Criando a primeira rota, no caso, a raiz da aplicacao
# Quando criamos rotas seguidas uma da outra, elas acessam a 
# mesma requisicao HTTP.
@app.route("/")
@app.route("/index")

# Criamos uma funcao com mesmo nome da rota para manter padrao

def index():
    return render_template("index.html")
